**Phase 1, single H200, no TP, no paged KV.** HuggingFace `Qwen3_5MoeForCausalLM` (text branch of the multimodal checkpoint) loaded in BF16 with `flash-linear-attention` for the 30 Gated DeltaNet layers and `causal_conv1d` for the depthwise conv path. A static microbatcher at the engine level drains a request queue and runs one batched `model.generate(input_ids:[B,L])` per cycle, left-padded, EOS-trimmed. Greedy/sampled requests are split into separate sub-batches because HF's `do_sample` is per-call. The whole thing is one process on one GPU; everything Phase 2 changes (TP=8, paged KV, continuous batching, FA2 paged decode) sits behind this engine boundary.
